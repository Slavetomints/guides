# hashID hashID is one of the quickest and easiest ways to determine what type of [hash](https://slavetomints.githuhb.io/guides/guides/vocabulary#hash) you are working with. For this guide, we'll be using `hashes.txt` that look like the following: ``` d8578edf8458ce06fbc5bb76a58c5ca4 e10adc3949ba59abbe56e057f20f883e 5f4dcc3b5aa765d61d8327deb882cf99 9686c6f3173c9d463fe3a78da4f3616c 0d107d09f5bbe40cade3de5c71e9e9b7 ``` So, how do we find the type of hash? Well, all we need to do is call `hashid [file]` like so. ``` ┌─[slavetomints@parrot]─[~] └──╼ $hashid hashes.txt --File 'hashes.txt'-- Analyzing 'd8578edf8458ce06fbc5bb76a58c5ca4' [+] MD2 [+] MD5 [+] MD4 [+] Double MD5 [+] LM [+] RIPEMD-128 [+] Haval-128 [+] Tiger-128 [+] Skein-256(128) [+] Skein-512(128) [+] Lotus Notes/Domino 5 [+] Skype [+] Snefru-128 [+] NTLM [+] Domain Cached Credentials [+] Domain Cached Credentials 2 [+] DNSSEC(NSEC3) [+] RAdmin v2.x Analyzing 'e10adc3949ba59abbe56e057f20f883e' [+] MD2 [+] MD5 [+] MD4 [+] Double MD5 [+] LM [+] RIPEMD-128 [+] Haval-128 [+] Tiger-128 [+] Skein-256(128) [+] Skein-512(128) [+] Lotus Notes/Domino 5 [+] Skype [+] Snefru-128 [+] NTLM [+] Domain Cached Credentials [+] Domain Cached Credentials 2 [+] DNSSEC(NSEC3) [+] RAdmin v2.x Analyzing '5f4dcc3b5aa765d61d8327deb882cf99' [+] MD2 [+] MD5 [+] MD4 [+] Double MD5 [+] LM [+] RIPEMD-128 [+] Haval-128 [+] Tiger-128 [+] Skein-256(128) [+] Skein-512(128) [+] Lotus Notes/Domino 5 [+] Skype [+] Snefru-128 [+] NTLM [+] Domain Cached Credentials [+] Domain Cached Credentials 2 [+] DNSSEC(NSEC3) [+] RAdmin v2.x Analyzing '9686c6f3173c9d463fe3a78da4f3616c' [+] MD2 [+] MD5 [+] MD4 [+] Double MD5 [+] LM [+] RIPEMD-128 [+] Haval-128 [+] Tiger-128 [+] Skein-256(128) [+] Skein-512(128) [+] Lotus Notes/Domino 5 [+] Skype [+] Snefru-128 [+] NTLM [+] Domain Cached Credentials [+] Domain Cached Credentials 2 [+] DNSSEC(NSEC3) [+] RAdmin v2.x Analyzing '0d107d09f5bbe40cade3de5c71e9e9b7' [+] MD2 [+] MD5 [+] MD4 [+] Double MD5 [+] LM [+] RIPEMD-128 [+] Haval-128 [+] Tiger-128 [+] Skein-256(128) [+] Skein-512(128) [+] Lotus Notes/Domino 5 [+] Skype [+] Snefru-128 [+] NTLM [+] Domain Cached Credentials [+] Domain Cached Credentials 2 [+] DNSSEC(NSEC3) [+] RAdmin v2.x --End of file 'hashes.txt'-- ``` So it looks like there were multiple hits for each hash, which makes sense, as some have overlapping formats, but the first few are the ones we're looking for as they are the most likely formats. Since [MD5](https://slavetomints.github.io/guides/guides/vocabulary#md5) is pretty common, we can safely assume that they are MD5 hashes. So now that we know what the format of the hashes are, how do we utilize that? This is where the `-m` and `-j` flag come into use. The `-m` flag displays the corresponding `hashcat` modes for the hashes.
